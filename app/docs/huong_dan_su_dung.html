📖 Hướng dẫn sử dụng

Phần mềm Tải Hóa Đơn Gmail — tự động tải hóa đơn điện tử (PDF + XML) từ hộp thư Gmail và sắp xếp vào thư mục theo từng khách.

1. Phần mềm này làm gì?

  • Quét hộp thư Gmail, tìm email hóa đơn điện tử của các nhà cung cấp.
  • Tải file đính kèm, hoặc tự vào link tra cứu của nhà mạng để tải PDF + XML.
  • Tự xếp file vào thư mục theo từng khách / từng người gửi mà bạn đã cấu hình.

2. Lần đầu sử dụng: thêm tài khoản Gmail

Vào thẻ Cấu hình & Bộ lọc:

  1. Bấm ➕ Thêm khách (Gmail).
  2. Nhập Mã khách (không dấu, ví dụ GP2), Tên hiển thị, Email.
  3. Chọn file credentials .json (file Google cấp để cho phép đọc Gmail).
    Chưa có file này? Xem Phụ lục: Cách tạo file credentials .json ở cuối hướng dẫn.
  4. Chọn Thư mục mặc định để lưu hóa đơn.
  5. Bấm Đăng nhập Google & Lưu → cửa sổ trình duyệt mở ra, chọn đúng email và cấp quyền.
Lần đầu cần đăng nhập Google một lần. Sau đó phần mềm nhớ đăng nhập; khi nào hết hạn nó sẽ tự mở trình duyệt để bạn đăng nhập lại.

3. Cấu hình thư mục lưu & bộ lọc

Trong thẻ Cấu hình & Bộ lọc, chọn khách rồi thiết lập:

  • Thư mục mặc định: nơi lưu hóa đơn cho email chưa có trong bộ lọc. Có thể dùng {date} = ngày hôm nay.
  • Bộ lọc theo người gửi: bấm ➕ Thêm để chỉ định "email người gửi này → lưu vào thư mục riêng". Bấm đúp một dòng để sửa nhanh.

Xong nhớ bấm 💾 Lưu cấu hình khách.

4. Tải hóa đơn

Vào thẻ Tải hóa đơn:

  1. Tích chọn một hoặc nhiều khách (có nút Chọn tất cả / Bỏ chọn).
  2. Chọn Chế độ tải:
    • Theo cấu hình — dùng khoảng ngày & điều kiện đã lưu cho khách.
    • Tải tất cả — quét mọi email khớp.
    • Chỉ chưa đọc — chỉ email chưa đọc.
  3. (Tùy chọn) tích Ghi đè khoảng ngày rồi chọn Từ ngày / Đến ngày.
  4. Bấm 🚀 Tải hóa đơn. Theo dõi tiến trình ở khung Nhật ký bên dưới.
Muốn xem trước phần mềm sẽ tìm email theo điều kiện gì mà chưa tải thật? Bấm Xem QUERY (thử).

6. Cập nhật phần mềm

Phần mềm tự kiểm tra bản mới khi mở. Bạn cũng có thể bấm 🔄 Kiểm tra cập nhật ở góc trên bên phải. Khi cập nhật, cấu hình và dữ liệu của bạn được giữ nguyên.

📎 Phụ lục: Cách tạo file credentials .json từ Google

File credentials.json là "giấy phép" để phần mềm được quyền đọc Gmail của bạn. Bạn chỉ cần tạo một lần, hoàn toàn miễn phí. Làm theo các bước sau:

Nên đăng nhập sẵn bằng chính tài khoản Gmail mà bạn muốn tải hóa đơn trước khi bắt đầu. Giao diện Google có thể thay đổi đôi chút theo thời gian, nhưng tên các mục vẫn gần giống như dưới đây.

Bước 1 — Mở Google Cloud Console

  1. Vào địa chỉ: https://console.cloud.google.com/
  2. Đăng nhập bằng tài khoản Gmail cần dùng. Nếu được hỏi, đồng ý Điều khoản dịch vụ.

Bước 2 — Tạo một dự án (Project)

  1. Bấm vào ô chọn dự án ở thanh trên cùng (cạnh chữ "Google Cloud").
  2. Bấm New Project / Dự án mới, đặt tên bất kỳ (ví dụ Tai Hoa Don) rồi bấm Create / Tạo.
  3. Chờ vài giây, rồi chọn đúng dự án vừa tạo ở ô trên cùng.

Bước 3 — Bật quyền đọc Gmail (Gmail API)

  1. Trong ô tìm kiếm trên cùng, gõ Gmail API rồi chọn kết quả "Gmail API".
  2. Bấm nút Enable / Bật.

Bước 4 — Khai báo màn hình xin quyền (OAuth consent screen)

  1. Vào menu trái: APIs & Services → OAuth consent screen (một số bản ghi là Google Auth Platform → Branding/Audience).
  2. Chọn loại External / Bên ngoài, bấm Create / Tạo.
  3. Điền các mục bắt buộc (có dấu *): Tên ứng dụng (ví dụ Tai Hoa Don), Email hỗ trợ người dùng = email của bạn, Email liên hệ nhà phát triển = email của bạn. Các mục khác để trống. Bấm Save and Continue / Lưu và tiếp tục qua các bước (không cần thêm Scopes).
Rất quan trọng: ở mục Test users / Người dùng thử nghiệm, bấm + Add Users và nhập chính địa chỉ Gmail của bạn, rồi Lưu. Nếu bỏ qua bước này, khi đăng nhập sẽ bị Google chặn với thông báo "đã chặn quyền truy cập".

Bước 5 — Tạo & tải file credentials .json

  1. Vào menu trái: APIs & Services → Credentials (hoặc Clients).
  2. Bấm + Create Credentials → OAuth client ID.
  3. Ở mục Application type / Loại ứng dụng, chọn Desktop app / Ứng dụng máy tính. Đặt tên bất kỳ, bấm Create.
  4. Hiện ra cửa sổ thành công — bấm Download JSON / Tải JSON để lưu file về máy. (Nếu lỡ tắt, bấm biểu tượng ⬇ tải xuống ở dòng client vừa tạo.)
File tải về có tên dạng client_secret_....json. Đây chính là file credentials .json cần chọn ở Bước 3 của mục 2 phía trên. Bạn có thể đổi tên file cho dễ nhớ (ví dụ credentials_GP2.json) — không sao cả.

Khi đăng nhập gặp màn hình cảnh báo "Google chưa xác minh ứng dụng"

Vì đây là ứng dụng riêng của bạn nên Google sẽ hiện cảnh báo. Cứ làm như sau là an toàn:

  1. Bấm Advanced / Nâng cao.
  2. Bấm Go to … (unsafe) / Chuyển đến … (không an toàn).
  3. Bấm Continue / Tiếp tục để cấp quyền đọc Gmail cho phần mềm.
Một file credentials .json có thể dùng chung cho nhiều tài khoản Gmail. Mỗi khi thêm khách mới, bạn chọn lại cùng file đó cũng được; phần mềm sẽ tạo đăng nhập riêng cho từng tài khoản.
